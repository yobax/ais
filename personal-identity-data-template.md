# 情報の3層構造

## Raw Identity

本人に関する原資料、事実、会話、写真、音声、作品、記録を可能な限りそのまま保持する層。

### 含まれるもの

- 元の会話
- 写真
- 音声
- 動画
- メール
- 日記
- 公的記録
- 作品
- 証言
- 未整理のメモ

---

## Structured Identity

Raw Identityを、時系列、人物、価値観、感情、環境などに分類・関連付けした層。

### 主な処理

- 分類
- 時系列化
- 人物との関連付け
- 出来事との関連付け
- 矛盾の記録
- 出典の記録
- 確度の付与
- Memory Weightの付与
- 更新履歴の保持

---

## Active Personalization

現在の会話でAIが即座に参照する、高密度に圧縮されたIdentity層。

### 役割

- 本人の現在地を理解する
- 応答方針を調整する
- 関連する記憶へ接続する
- 必要に応じてStructured IdentityやRaw Identityを参照する
- 本人の価値観、環境、目標、関係性に沿って応答する

Active Personalizationは情報を捨てる層ではなく、全情報へ接続する索引として扱う。

---

# 情報管理用メタデータ

各情報には、可能な限り次の属性を付与する。

| 属性 | 説明 |
|---|---|
| `source` | 情報の出典 |
| `recorded_at` | 記録日 |
| `event_date` | 出来事が起きた日 |
| `verified_at` | 最終確認日 |
| `confidence` | 情報の確度 |
| `memory_weight` | 本人理解における重要度 |
| `status` | current / past / uncertain / corrected |
| `related_people` | 関連人物 |
| `related_places` | 関連場所 |
| `related_projects` | 関連プロジェクト |
| `sensitivity` | 情報の扱い区分 |
| `notes` | 補足、矛盾、未確認事項 |

---

# 基本原則

1. 本人に関する情報は、原則としてすべてIdentityの一部として扱う。
2. 分類は削除のためではなく、理解と参照精度を高めるために行う。
3. 成功、失敗、矛盾、感情、身体、環境を分離せず記録する。
4. 事実と本人の解釈を区別する。
5. 情報には出典、日時、確度、重要度を付与する。
6. 古い情報を消さず、変更履歴として残す。
7. AIによる推測は、本人が語った事実と明確に分ける。
8. Active Personalizationは全情報の代替ではなく、全情報への入口とする。
9. 人物理解は固定プロフィールではなく、時間と対話によって更新されるものとして扱う。
10. 本人の意思、価値観、環境、関係性を統合して理解する。
